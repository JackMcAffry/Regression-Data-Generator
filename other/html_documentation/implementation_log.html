class implementation_log.log_generator.Log_Chunk_Generator[source]¶

Bases: abstract_classes.dgp_generator.DGP_Chunk_Generator

Implementation of DGP_Generator, used for Logistic Regression. Its functions generate dataset chunk by chunk with optimization level, for memory saving

generate_dataset(params, saver, chunks=1, verbose=False, opt_level='float64', compression=False, compr_type=None)[source]¶
Parameters

  • params – instance of Log_Params

  • chunks – int, number of chunks that the dataset will have. It represents the number of total iterations

  • verbose – verbose output

  • opt_level – str that represent the float type for casting arrays. Can be ‘float64’, ‘float32’ or ‘float16’. Default ‘float64’

‘float64’ -> default (no optimization)

‘float32’ -> downcast to float32

‘float16’ -> downcast to float16

Parameters

  • compression – bool. Set true if you want compression (must not have compr_type = None)

  • compr_type – str that represent the compression type. Works only if compression is set to True. Can be 3 values:

‘gz’ -> using gunzip compression, file will be saved as <filename>.csv.gz

‘bz’ -> using bzip compression, file will be saved as <filename>.csv.bz

‘xz’ -> using xz compression, file will be saved as <filename>.csv.xz

You may not assign this method to a variable, because it doesn’t mantain anything in memory
